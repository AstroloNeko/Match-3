# 订单货架三消原型

一个零依赖 H5 小游戏原型，用来验证“货架遮挡 + 暂存槽三消 + 顾客订单”的核心循环。

## 玩法

- 屏幕上方有 3 个订单，每个订单需要一种货物凑满 3 个。
- 点击可选货物，货物会进入下方暂存槽。
- 暂存槽中三个同类货物会自动出货。
- 如果出货类型匹配订单，就完成 1 单并补新订单。
- 上层货物会遮挡周围下层货物，被挡住的货物不能选择。
- 每关完成目标订单数即可通关。
- 暂存槽满或倒计时结束则失败。
- “卡槽+1”按钮模拟激励视频广告奖励，每关默认 7 格，最多扩到 9 格。

## 可控随机

这版不做完全随机：

- 开局会优先生成当前订单需要的货物。
- 完成订单后会围绕新订单补货。
- 如果当前可选货物无法服务订单或暂存槽中的对子，系统会暗中把一个可选货物改成有用类型。
- 这样玩家仍然会失败，但失败更像是决策和卡槽管理问题，而不是纯随机死局。

## 运行

直接用浏览器打开 `index.html` 即可。也可以用本地静态服务打开：

```powershell
python -m http.server 5173
```

然后访问 `http://localhost:5173`。
