# 订单货架三消原型

一个零依赖 H5 小游戏原型，用来验证“货架遮挡 + 暂存槽三消 + 顾客订单”的核心循环。

## 玩法

- 屏幕上方有 3 个订单，点击可选货物进入下方暂存槽。
- 暂存槽中三个同类货物会自动出货。
- 出货类型匹配订单时，会推进订单进度；订单完成后补新订单。
- 上层货物会遮挡周围下层货物，被挡住的货物不能选择。
- 每关完成目标订单数即可通关。
- 暂存槽满或倒计时结束则失败。
- “卡槽+1”按钮模拟激励视频广告奖励，每关默认 7 格，最多扩到 9 格。

## 难度曲线

关卡不再只是加一点数量，而是逐步引入新的压力：

- 第 1 关：教学关，小货架、少种类、普通单，基本让玩家熟悉规则。
- 第 2 关：难度急剧上升，货架扩到 8x10，急单、大单、双品类、冰冻货和捆绑货都会开始出现。
- 后续关卡：货架固定为移动端更易辨认的 8 列，并逐步纵向扩到 14 行；货物种类、目标订单数、货架密度和遮挡比例都会显著增加。
- 难度曲线更陡：复杂订单更早出现，急单耐心更短，高关卡补货更密、导演兜底更弱。
- 连续完成订单会给连单奖励；清出不匹配订单的散货会扣一点时间。

## 特殊货物

- 星标货：每件参与订单出货都会立即奖励 3 秒，并有明显的金色标记。
- 冰冻货：进入暂存槽后经历三消会变成冷链货；冷链货参与出货会奖励时间，并连锁融化货架上的一件冰冻货。
- 炸弹道具：不属于任何商品种类，会进入暂存槽上方的工具位，不计入满槽死亡容量，拖到另一张卡上会把目标退回货架。
- 捆绑货：点其中一个会把另一个一起拉进暂存槽；若两端各自都凑齐三消，会自动触发两张订单连锁双发。

## 开始界面

游戏启动时会先显示开始界面，可以打开玩法教程查看订单、遮挡和特殊货物说明。
第 1 关通关后会出现“难度升级”过场，再进入断崖式加压的第 2 关。

## 可控随机

这版仍然不是完全随机：

- 开局会优先生成当前订单需要的货物。
- 完成订单后会围绕新订单补货。
- 如果当前可选货物无法服务订单或暂存槽中的对子，系统会按关卡难度决定是否暗中修正一个可选货物。
- 关卡越往后，导演系统越少兜底，玩家的卡槽管理会更重要。

## 关卡种子与诊断

每局会生成一个局种子，每关再由“局种子 + 关卡号”派生独立随机序列。点击游戏下方的种子栏可复制带 `seed` 和 `level` 参数的复现链接；同一链接会生成相同的订单、牌堆、特殊货、遮挡和捆绑关系。

诊断面板会记录尾货清仓、残局融冰、无可点击货物、订单不接收现有三消、库存不足和失败状态。复制复现信息可同时取得种子、当前库存、订单需求和最近诊断事件。

关卡生成会先按初始订单反向组织三元组，再将订单骨架优先放入可剥离层。炸弹作为额外工具加入，不再占用商品库存。布局完成后会模拟 9 格卡槽、捆绑进槽、冻结与融冰、订单限制和尾货规则；失败时只撤销模拟卡槽中造成问题的链条或冰冻状态，最多尝试 12 个候选，并把最终清除路径证书写入诊断报告。后续补单会优先跟随证书中仍有库存的商品类型，同时保留卡槽对子救援的更高优先级。

种子与生成器批量测试：

```powershell
node tests\seed-smoke.js
```

## 本地排行榜

开始界面会显示历史最高分、最高到达关卡和最近一次成绩。纪录保存在浏览器本地 `localStorage` 中，不需要服务器。

## 固定牌堆清场

当前版本改为每关开局生成固定牌堆，出货后不再自动补货。订单会尽量从剩余牌堆库存中下发，只有当前订单需要的三消才会清除；没有订单的三张同类货会暂时卡在暂存槽里。这样保留订单机制，同时让货架产生越清越少的视觉反馈。

炸弹每关最多 3 个，并在工具位折叠显示为数量，避免道具栏溢出。
炸弹现在用于腾出暂存槽：拖到目标卡上会把目标退回货架，而不是永久销毁货物，避免固定牌堆被炸到无法通关。

冰冻货仍然优先通过正常三消推进解冻；如果残局中当前订单已经没有足够的非冰冻牌可以继续推进，系统会触发“残局融冰”，按冷却节奏自动降低暂存槽冰冻层数，避免锁死。

当前版本默认 4 个同时订单、9 个暂存槽，激励扩槽最多到 11 个。冰冻货只需要一次三消推进即可解冻。

通关目标改为清空货柜。订单负责限制哪些三消可以出货，订单完成数只作为表现记录；当残局只剩无法组成三消的散货时，会自动打包清仓，保证视觉上真正清空。
